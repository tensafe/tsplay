# Lesson 134 Review Checklist

把一个教程示例拿来 review 时，至少先看这几件事：

1. `name` 能不能直接说明 lesson 编号和任务意图。
2. `description` 是不是在讲用户结果，而不是只说“做点什么”。
3. `save_as` 有没有表达出变量角色，而不是 `x`、`tmp`、`result2`。
4. artifact 路径是不是落在 lesson 自己的目录里。
5. 输出文件名是不是人能猜到内容。
6. 这条 Flow 能不能在不读源码实现的情况下，大概看懂执行意图。
7. 如果今天需要交给别人接手，最可能让别人困惑的点是什么。
