# 高级教程

高级教程的重心，是“系统化交付”。

从这一层开始，不再只关心“这条 Flow 能不能跑”，而是要同时回答：

- 它能不能被团队长期维护
- 它能不能被审查和发布
- 它能不能被打包交给别人直接用
- 它能不能作为一套课程继续生长

## 适合谁

- 要在团队里推广 TSPlay
- 要做集成、评审、交付、培训、规范
- 要把 `tsplay` 作为产品能力或交付能力的人

## 高级阶段的核心主题

### 1. 安全边界

要系统理解：

- `allow_lua`
- `allow_http`
- `allow_file_access`
- `allow_redis`
- `allow_database`
- `allow_browser_state`

重点不是记参数，而是知道为什么要分这些边界。

### 2. 大型 Flow 的组织方式

要开始建立规范：

- 步骤命名
- 变量命名
- artifact 目录约定
- 环境变量约定
- review 规则

### 3. 二进制发布与交付包

这一层要理解为什么现在要把：

- `docs/`
- `script/`
- `demo/`
- `ReadMe.md`

一起打包进二进制。  
这不是“方便一点”，而是把教程、示例、运行入口做成同一个交付物。

### 4. 课程持续演进

高级阶段不应该只停在“我会用了”。  
还要会：

- 补新主题
- 重构旧文档
- 按层级安排学习顺序
- 维护一条不会跳跃的增长主线

## 高级阶段的交付物

- 一份团队规范草案
- 一份教程演进计划
- 一组可交付的示例包
- 一套“新人从哪里开始学”的推荐顺序

## 高级阶段的退出标准

- 能独立组织一套可交付教程
- 能评审 Flow 的结构质量而不只是看它能不能跑
- 能解释 TSPlay 的安全边界和交付边界
- 能继续按同一逻辑把教程往下演进
